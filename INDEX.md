## 🎓 **Key Concepts to Know**

### **Feature-Based Architecture**
- Code organized by feature, not by type
- Each feature has own components, hooks, api
- Easy to add new features
- Better code organization

### **Custom Hooks**
- `useDownload` - Handles download logic
- Reusable across components
- Encapsulates state management

### **CSS Modules**
- Scoped component styles
- No naming conflicts
- Better organization
- One .css per component

### **Service Layer**
- `downloadService` - API communication
- Centralized API calls
- Easy to test and mock
- Reusable across app
